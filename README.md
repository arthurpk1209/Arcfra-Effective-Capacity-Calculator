# Arcfra-Effective-Capacity-Calculator

| 規格項目                       | 實作說明                                                                                 |
|----------------------------|--------------------------------------------------------------------------------------|
| 頂部提示                   | 藍色橫幅明確標示「本工具僅適用於安裝 ACOS 在硬體的 RAID 1 的配置上」。                                           |
| 4.1.1 / 4.1.2 Cache 計算 | TB 級硬碟 `×0.909495`，GB 級硬碟 `×0.000909495`，統一扣除 `20/1024` TiB。                         |
| 4.2 Capacity 計算       | 所有 Capacity Disk 先換算為 GB 總和，再 `×0.000909495`，最後扣除 `5.85%` 損耗。                        |
| 4.3.1 冗餘 Cache 比例     | 每節點檢查 `(CacheUsable / 2) / CapacityUsable ≥ 1/10`。未達標的節點卡片會顯示**紅色邊框**，並在全局結果區列出詳細告警。 |
| 4.3.2 單顆 Cache 警告     | 若節點僅配置 1 顆 Cache Disk，該卡片顯示**黃色邊框**，並觸發「系統 Cache 沒有冗餘」的全局警告。                         |
| 4.4.1 / 4.4.2 RF 計算   | 嚴格依公式：`每節點Capacity可用空間 × (節點數−1) / 2`（RF2）與 `/3`（RF3）。                               |
| 4.5 / 4.6 保留策略        | 儲存空間保留（預設 20%）與快照空間保留（預設 10%），可獨立勾選與調整百分比，最終有效空間自動乘算對應係數。                            |
