# Arcfra-Effective-Capacity-Calculator

Prompte
請參考以下網頁，製作一個Arcfra Effective Capacity
https://kauteetech.github.io/vsancapacity/hybrid 

最上方提示為：本工具僅適用於安裝ACOS再硬體的RAID1的配置上
1. 節點形式選項：分為Hybrid和All Flash
2. 如果節點是Hybrid會有Cache Disk和Capacity Disk選項
2.1 Cache Disk，可以有選單選擇硬碟配置，容量選項為960GB/1TB/1.6TB/1.92TB/2TB/3.2TB/3.84TB/4TB/6.4TB/7.68TB/8TB/12.8TB/15.36TB
2.2 Capacity Disk, 可以有選單選擇硬碟配置，容量選項為1TB/1.2TB/1.8TB/2TB/2.4TB/4TB/6TB/8TB/10TB/12TB/14TB/16TB/18TB
3. 節點數量選項：每個節點可以有不同Cache Disk和Capacity Disk的配置
4. Hybrid計算方式
4.1.1 每個節點Cache可用空間：所有的Cache Disk容量加總(如果是TB的硬碟)*0.909495-(20/1024)
4.1.2 每個節點Cache可用空間：所有的Cache Disk容量加總(如果是GB的硬碟)*0.000909495-(20/1024)
4.1.3. 每個節點Capacity可用空間：(所有的Capacity Disk容量加總*0.000909495)-(所有的Capacity Disk容量加總*0.000909495)*0.0585
4.2.1 比例要求：每個節點冗餘Cache可用空間＝每個節點Cache可用空間/2：每個節點Capacity可用空間 > 1:9，如果沒有則告警不能設定
4.2.2 如果每個節點只設定一個Cache Disk則產生"系統Cache沒有冗餘的告警"
5. All Flash計算方式
5.1 All Flash只能選同一種容量的硬碟，不能選不同容量的硬碟
5.2.1 每個節點Capacity可用空間：所有的Capacity Disk容量加總*0.909495(如果是TB的硬碟)
5.2.2 每個節點Capacity可用空間：所有的Capacity Disk容量加總*0.000909495(如果是GB的硬碟)
6. 整體空間展示
6.1.1 整體系統顯示可用空間(RF2)：每個節點Capacity可用空間*(節點數)/2
6.1.2 整體系統顯示可用空間(RF3)：每個節點Capacity可用空間*(節點數)/3
6.2.1 整體有效可用空間(N+1和RF2)：每個節點Capacity可用空間*(節點數-1)/2
6.2.2 整體有效可用空間(N+1和RF3)：每個節點Capacity可用空間*(節點數-1)/3
7.1 新增儲存空間容量保留選單：預設為20%，表示需要保留總容量的20%空間，客戶可以依照需求調整，如果客戶勾選了此選單則總有效空間因為保留政策而下降
7.2 新增快照空間容量保留選單：預設為10%，表示需要保留總容量的10%空間，客戶可以依照需求調整，如果客戶勾選了此選單則總有效空間因為保留政策而下降



| 規格項目                           | 實作說明                                                                 |
| ------------------------------ | -------------------------------------------------------------------- |
| **頂部提示**                       | 藍色橫幅標示「本工具僅適用於安裝 ACOS 在硬體的 RAID 1 的配置上」。                             |
| **節點形式切換**                     | 全局設定區提供 **Hybrid** / **All Flash** 切換按鈕，切換後整體介面與計算邏輯即時變更。            |
| **Hybrid 模式**                  | 每個節點獨立卡片，可各自配置 Cache Disk（13 種 GB/TB 規格）與 Capacity Disk（13 種 TB 規格）。 |
| **All Flash 模式**               | 全局統一硬碟規格選擇（所有節點強制相同），每節點盤數在全局設定，節點卡片為只讀匯總。                           |
| **Hybrid Cache 計算 (4.1)**      | GB 硬碟 `×0.000909495`、TB 硬碟 `×0.909495`，統一扣除 `20/1024` TiB。           |
| **Hybrid Capacity 計算 (4.1.3)** | 總和轉 GB 後 `×0.000909495`，再扣除 `5.85%` 損耗。                              |
| **冗餘比例檢查 (4.2.1)**             | 每節點驗證 `(Cache可用 ÷ 2) / Capacity可用 ≥ 1/9`，未達標節點顯示紅框並全局告警。             |
| **單碟冗餘警告 (4.2.2)**             | 僅配置 1 顆 Cache Disk 的節點顯示黃框，並觸發「系統 Cache 沒有冗餘」警告。                     |
| **All Flash 計算 (5.2)**         | 全局規格 × 盤數 × 係數（GB: `0.000909495`、TB: `0.909495`）。                    |
| **系統顯示空間 (6.1)**               | `每節點Capacity × 節點數 / RF`，反映系統層面原始可用容量。                               |
| **有效可用空間 N+1 (6.2)**           | `每節點Capacity × (節點數−1) / RF`，反映容錯後實際可用容量。                            |
| **儲存保留 (7.1)**                 | 預設 20%，僅套用於**有效可用空間**（N+1），可勾選啟閉與調整百分比。                              |
| **快照保留 (7.2)**                 | 預設 10%，同樣僅套用於**有效可用空間**，兩項保留策略採連乘方式疊加。                               |
